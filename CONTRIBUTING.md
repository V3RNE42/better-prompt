# Contributing

Thanks for wanting to make this better! Here's how:

## Opening Issues

- Use a clear, descriptive title
- Include the LLM, context, and prompt pattern you're working with
- If reporting a bug or limitation, include before/after examples

## Pull Requests

1. Fork the repo
2. Create a branch: `git checkout -b feature/my-change`
3. Make your changes
4. Update the README or SKILL.md if needed
5. Submit a PR with a clear description of what and why

## Style

- Keep it concise. This is a reference, not a novel.
- Examples > explanations. A good before/after is worth a paragraph.
- Data beats opinions — if you cite a paper or benchmark, link it.

## Code of Conduct

Be excellent to each other. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
